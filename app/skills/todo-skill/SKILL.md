# Todo Skill

管理复杂任务的进度跟踪，支持依赖关系管理。

## 何时触发

| 场景 | 判断标准 |
|------|---------|
| 复杂任务 | 任务需要 3+ 个步骤完成 |
| 多步骤工作 | 涉及多个阶段的开发工作 |
| 进度跟踪 | 用户要求跟踪任务进度 |

**简单任务不需要创建 Todo**（如"帮我打印 hello world"）。

## 核心概念

| 概念 | 说明 |
|------|------|
| Todo 文件 | 每个任务一个 .md 文件 |
| 状态 | 待办(Todo) / 进行中(In Progress) / 已完成(Done) |
| 依赖关系 | 步骤间的先后顺序 |
| 阻塞任务 | 等待依赖完成的步骤 |

## 文件结构

```
todo/                           # 存储目录
├── INDEX.md                    # 所有 Todo 索引
└── YYYY-MM-DD-HHMMSS-title.md  # 任务文件
```

## 依赖关系设计

### 两种依赖类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `依赖` | 必须等待前置完成 | 步骤B 依赖 步骤A |
| `阻塞` | 完成后才能启动下游 | 登录功能 阻塞 个人中心 |

### 依赖链示例

```
用户登录功能
├── 1. 数据库设计
├── 2. 后端 API（依赖 数据库设计）
└── 3. 前端页面（依赖 后端 API）
    │
    ↓
4. 集成测试（依赖 前端页面）
```

## Todo 文件格式

```markdown
# 任务名称

## 元信息

| 字段 | 值 |
|------|-----|
| 优先级 | 高/中/低 |
| 创建时间 | YYYY-MM-DD HH:MM:SS |
| 状态 | 进行中 / 已完成 |
| 进度 | 40% |

## 依赖关系

| 步骤 | 依赖 | 说明 |
|------|------|------|
| 步骤2 | 步骤1 | 步骤2 等待 步骤1 完成 |
| 步骤4 | 步骤2, 步骤3 | 步骤4 等待 步骤2和3完成 |

## 任务步骤

### 待办 (Todo)
- [ ] 步骤1

### 进行中 (In Progress)
- [ ] 步骤2（阻塞中：等待 步骤1）

### 已完成 (Done)
- [x] 步骤3

## 备注
额外说明或上下文
```

## 操作流程

### 1. 创建 Todo

当判断任务复杂时：

```
1. 分析任务步骤和依赖关系

2. 创建 todo 文件
   scripts/todo_manager.py create "<任务名>" "<优先级>"

3. 分解任务为步骤

4. 标注步骤间的依赖关系
```

### 2. 执行策略

```
1. 找出无依赖的步骤（可立即执行）

2. 按顺序执行步骤

3. 依赖完成后，解除阻塞，启动下一步

4. 重复直到所有步骤完成
```

### 3. 更新进度

完成某步骤后：

```
1. 标记步骤为已完成
   - [ ] → - [x]

2. 检查是否有步骤解除阻塞

3. 重新计算进度百分比

4. 更新 INDEX.md
```

## INDEX.md 格式

```markdown
# Todo 索引

## 进行中

| 文件 | 任务 | 优先级 | 进度 | 阻塞 |
|------|------|--------|------|------|
| [title.md](title.md) | 任务名 | 高 | 40% | - |

## 阻塞中

| 文件 | 任务 | 等待 |
|------|------|------|
| [title.md](title.md) | 任务名 | task-a.md |

## 已完成

| 文件 | 任务 | 完成时间 |
|------|------|----------|
| [title.md](title.md) | 任务名 | YYYY-MM-DD |
```

## 工具脚本

详见 `scripts/todo_manager.py`：

- `create <name> <priority>` - 创建新 Todo
- `add-step <filename> <step>` - 添加步骤
- `update-step <filename> <step> <status>` - 更新步骤状态
- `list` - 列出所有 Todo
- `show <filename>` - 显示 Todo 内容
- `done <filename>` - 标记完成
- `delete <filename>` - 删除 Todo

## 注意事项

- **自动触发**：Agent 根据任务复杂度自动判断是否创建
- **诚实报告**：进度百分比必须基于实际步骤计算
- **及时更新**：完成步骤后立即更新状态
- **依赖明确**：每个步骤必须标注依赖关系
- **阻塞可视化**：在 INDEX.md 中清晰显示阻塞状态
